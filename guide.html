TITAN for AgRobotics App

Welcome to the TITAN for AgRobotics App — a tool for programming your AgRobotics robot right in your browser. No installation required. This guide walks you through everything you need to get started, write your first program, and confidently use the app on competition day.

What Is This App?

The TITAN for AgRobotics App lets you write programs for your robot using two methods:

  • Block Coding — drag and drop colorful puzzle pieces to build programs visually. Great for beginners.
  • Python Coding — write MicroPython code directly in a full editor with syntax highlighting.

Both methods send the same type of program to your robot — an titan.* MicroPython program that controls motors, reads sensors, plays sounds, and lights up pixels.

Tip: You can switch between Block and Python tabs at any time. The block editor even shows you the Python code it generates in the MicroPython panel — a great way to start learning Python!

Opening the App

If you downloaded the app as a zip file:

Unzip the agrobotics-app.zip file into a folder on your computer.
Open the agrobotics-app folder and double-click agrobotics-app.html.
Chrome or Edge will open the app. You may see a browser message about local files — this is normal. Click through to continue.
Note: Always open the app in Chrome or Edge on Windows, Mac, or Chromebook. On iPad or iPhone, use the free Bluefy browser from the App Store instead — it supports all app features including Bluetooth.

The Workspace

The workspace is where you build your program. It changes depending on which type of tab you have open.

Block Coding Workspace BLK

The block workspace has two parts:

  • Toolbox — the panel on the left with all the block categories (Control, Movement, Sensors, etc.). Click a category to expand it, then drag a block into the workspace.
  • Canvas — the large area where you arrange and connect blocks. Scroll to zoom in/out. Click and drag the background to pan around.
Tip: Blocks snap together like puzzle pieces. A block will only connect where it fits — the shape tells you what kind of value it accepts.

Python Coding Workspace PY

The Python workspace is a full code editor with syntax highlighting, line numbers, and auto-indent. You write MicroPython code directly using titan.* functions — the same functions that the block editor generates. The toolbox is hidden in Python mode since you type your code instead.

The Right Panel

The right panel has three tabs:

TabWhat It Shows
⚙ MicroPythonThe Python code that your blocks generate, updated live as you work. Great for seeing how blocks translate to real code. Hidden when a Python tab is active.
▶ ConsoleShows everything printed by the robot — sensor readings, status messages, errors, and any Print blocks in your program. Type commands directly in the input bar to send them to the robot.
📖 ReferenceA quick block reference you can browse without leaving the app. For the full reference, use Help → Block Reference.

Connecting to Your Robot

Connecting via Bluetooth (BLE)

Power on your robot. Wait for the startup beep and the LED to turn on.
Click Connect in the toolbar. The connection dialog opens.
Choose Bluetooth (BLE). Your browser will show a list of nearby Bluetooth devices.
Select your robot from the list — it will appear as something like AgRobot-xxxx.
Click Pair. The Connect button turns green and the status shows Connected.
Important: Bluetooth works in Chrome or Edge on Windows, Mac, and Chromebook. On iPad or iPhone, use the free Bluefy browser (App Store) — it fully supports Web Bluetooth. If you don't see your robot in the list, make sure it is powered on and within range. Try moving closer.

Connecting via USB

Connect a USB data cable from your robot to your computer. Make sure it is a data cable, not just a charging cable.
Power on your robot.
Click Connect and choose USB / Serial. Select the correct COM port from the list.
The Connect button turns green when connected.

Enabling and Disabling Bluetooth

Your robot has a built-in Bluetooth on/off toggle — useful for competition days when you want to prevent accidental connections from other devices.

Hold Button B while powering on the robot to toggle Bluetooth on or off.
All four pixels flash to confirm the new setting:
●
Blue flash
= Bluetooth Enabled
●
Red flash
= Bluetooth Disabled
The setting is saved. Every time the controller powers on or resets, the pixels briefly flash blue (enabled) or red (disabled) as a reminder of the current state.
Note: Even with Bluetooth disabled, you can still connect and upload programs via USB.

Your First Block Program

Let's write a simple program that drives the robot forward for 30 cm and stops. Follow these steps:

Create a new tab. Click + New Tab in the tab bar. When the dialog appears, choose Block Coding and click Create Tab.
Add a Robot Config block. In the toolbox, click 🤖 Movement. Drag the 🤖 Robot Config block onto the workspace. This sets up your motors and wheel size. Leave the defaults: Left=1, Right=2, Diameter=80, Wheelbase=185.
Add Begin and End. Click 🌾 Control in the toolbox. Drag 🌾 Code Start to the top of your program and 🌾 Code End to the bottom. Every program needs these — Begin waits for you to press Select before starting, and End stops everything cleanly.
Add a Set Speed block. From Movement, add 🤖 Set Speed 50% after the Config block. This sets how fast the robot moves.
Add a Move block. From Movement, drag 🤖 Move Forward 30 cm Wait Yes into the program. Connect it after the Set Speed block.
Add a Stop block. Add 🤖 Stop Moving after the Move block.

Your program should look like this from top to bottom:

🌾 Code Start 🤖 Robot Config Left:1 Right:2 Diam:80 WB:185 🤖 Set Speed 50% 🤖 Move Forward 30 cm Wait:Yes 🤖 Stop Moving 🌾 Code End
Connect your robot (see Connecting section above).
Click Run in the toolbar. Your program is sent to the robot. Place it on a flat surface and press the Select button on the controller when prompted. The robot should drive forward 30 cm and stop!
Tip: Check the MicroPython panel on the right while you build — you can see exactly what code your blocks are generating. It updates live as you add blocks.

Your First Python Program

Python mode lets you write MicroPython code directly. Here's the same program as above — written in Python:

Create a Python tab. Click + New Tab and choose Python Coding. The editor opens with a starter template already filled in.
The editor already has: 
import titan
import time

titan.Begin()
# Write your MicroPython program here
# Use titan.* functions to control the robot
titan.End()
Add your code between Begin and End: 
import titan
import time

titan.Begin()

titan.setMovementMotors(1, 2)
titan.setMovementWheelDiameter(80)
titan.setMovementWheelbase(185)
titan.setMovementSpeed(50)
titan.moveForward(30, 'cm', True)
titan.stopMoving()

titan.End()
Click Run to send it to your robot. Press Select on the controller to start.
Tip: Use Ctrl+/ (or Cmd+/ on Mac) to comment or uncomment lines in the Python editor. Tab indents, Shift+Tab un-indents.

Using the Examples Browser

The app comes with 38 ready-to-use example programs covering every category of blocks. They're a great way to learn by seeing working code.

Click File → Examples in the toolbar.
Browse examples by category — Movement, Sensors, Motors & Servos, Buttons/Pixels/Sound, Diagnostics, and more.
Each example has two load buttons: 🧩 Blocks loads a block version, ⌨ Python loads the Python version. Click whichever you prefer.
The example opens in a new tab. You can run it as-is or modify it to experiment.
Tip: Start with Example 01 — First Move if you haven't run a program yet. Then try the sensor examples with a sensor plugged into port 1.

The Slot System

Your robot can store up to 16 programs in its memory — organized into 4 groups of 4 slots each. Programs stored in slots run without a computer, using just the buttons on the robot.

Groups are identified by both a letter and a color:

GroupColorSlotsProgram Numbers
A■ RedA-A, A-B, A-C, A-D1, 2, 3, 4
B■ GreenB-A, B-B, B-C, B-D5, 6, 7, 8
C■ BlueC-A, C-B, C-C, C-D9, 10, 11, 12
D■ YellowD-A, D-B, D-C, D-D13, 14, 15, 16
Color-blind friendly: Each group has both a letter (A/B/C/D) and a color. You can always use the letter instead of the color — they mean the same thing.

Uploading a Program to a Slot

Connect your robot via Bluetooth or USB.
Choose a slot. Click the SLOT button in the toolbar to open the slot picker. Occupied slots show a colored dot. Click the slot you want (e.g. A-A for Group A, first slot).
Click Upload to Slot. The program uploads to the controller. The launcher restarts automatically.
A colored pill appears on the tab showing which slot it's in — for example A-A means Group A (Red), Slot A.
Warning: Uploading to an occupied slot will overwrite whatever is already there. The app warns you before this happens. If you need to keep a program, save it to your computer first using File → Save Program.

Movement Blocks

All movement blocks require a 🤖 Robot Config block at the start of your program. This tells the app which motors are left and right, and how big your wheels are.

BlockWhat It Does
🤖 Robot Config (2WD)Sets left motor, right motor, wheel diameter, and wheelbase. Use once at program start. Default: Left=1, Right=2.
🤖 Robot Config (4WD)Same as 2WD but assigns two motors per side. Default: Left1=1, Left2=2, Right1=3, Right2=4.
🤖 Set Speed %Sets speed for all movement blocks. Range 0–100%. Set this before your first movement block.
🤖 Move Forward/BackwardMoves a set distance in cm or wheel rotations. Wait=Yes holds until complete.
🤖 Start MovingStarts moving continuously. Robot keeps going until Stop Moving runs.
🤖 Turn DegreesPivots the robot in place by the given number of degrees using wheel encoders.
🧭 Gyro TurnTurns a precise number of degrees using the gyro sensor. More accurate than encoder turns on slippery surfaces. Add-on required
🤖 Stop MovingStops both drive motors immediately.
Gyro Turn requires the optional Grove 6-Axis IMU Sensor (SKU 105020012) plugged into a sensor port. It's not included in the standard AgRobotics kit.

Saving Your Work

Always save your programs to your computer so you don't lose them. Use File → Save Program.

Tab TypeFile SavedWhat It Contains
BLK Blocks tab.xmlYour block layout — reloads exactly as you left it.
PY Python tab.pyYour Python code — opens directly in the Python editor.

The slot information is embedded in the saved file — when you reload it, the slot pill reappears on the tab automatically.

Tip: Name your files clearly — something like square-pattern-v2.xml is much easier to find later than untitled.xml. Double-click a tab name to rename it before saving.

Loading Programs

To reload a saved program, use File → Load Code. The file picker will open — find your saved .xml or .py file and select it. It opens in a new tab with the correct mode automatically (Blocks for .xml, Python for .py).

Note: When running the app locally from a zip file, the Examples browser loads from content built into the app itself — not from files on your computer. Your own saved files are always loaded using the file picker.

Troubleshooting

ProblemWhat to Try
Robot doesn't appear in the Bluetooth listMake sure the robot is powered on. Move closer. Check that Bluetooth isn't disabled (hold Button B at power-on to toggle). Refresh the browser and try again.
Bluetooth connected but nothing happens when I click RunCheck the Console tab for error messages. Make sure your program has an Code Start and Code End block.
Robot moves but in the wrong directionYour left and right motor assignments may be swapped. In the Robot Config block, try switching the Left and Right motor numbers.
Distance moves are inaccurateCheck that your Wheel Diameter and Wheelbase values in the Robot Config block match your actual robot measurements in mm.
Gyro turns are stopping short or overshootingThe gyro coast margin is tuned for 50% speed. If you changed speed, some variation is expected.
Sensor always reads 0 or gives errorsMake sure the sensor is plugged into the correct port and the port number in the block matches. Check the Console for error messages.
USB connection not workingTry a different USB cable — many cables only charge and don't carry data. Try a different USB port on your computer.
Program runs from Run but not from the launcherMake sure you clicked Upload to Slot — Run sends the program directly but doesn't save it. The launcher only runs programs saved to slots.
Robot won't stopPress the ■ Stop / Reset button in the toolbar. This immediately halts the program and returns to the launcher.

Quick Tips

  • Always start every program with 🌾 Code Start and end with 🌾 Code End.
  • Always put the 🤖 Robot Config block before any movement blocks.
  • Use Print blocks to print sensor values to the Console while your program runs — this is the best way to debug.
  • Use Run for quick testing. Use Upload to Slot when your program is ready for competition.
  • Save your programs to your computer regularly — the app doesn't store them between sessions.
  • The Stop / Reset button always works — use it any time the robot behaves unexpectedly.
  • On competition day, disable Bluetooth by holding Button B at power-on (you'll hear a low tone). This prevents accidental connections from other devices.
  • The Block Reference (Help → Block Reference) has details on every single block — keep it open in another tab while you work.
  • Load one of the 38 example programs to see how blocks connect together. Modify the example to learn how each part works.
